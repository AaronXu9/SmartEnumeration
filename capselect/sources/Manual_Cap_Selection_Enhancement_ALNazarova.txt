Introduction:
The selection of appropriate fragments having enough space for growth during enumeration into full
molecules allows improved enrichment in final docking. The selection algorithm represents an iterative
search of the possible vectors of growth which are represented by spheres. Available spheres with certain
radii are iteratively searched until either a clash into the wall of the pocket is observed or an exit out of the
pocket is detected. The center of the first sphere is defined by the coordinates of the labeled cap of the
fragment. The cap defines the point of the growth during enumeration. Apart from the single-labeled caps,
multiple labeled (as an aromatic ring) and double-labeled caps are also part of the fragment library.


Improved version of the CapSelection algorithm, CapSelection2 includes consideration of variable vectors

Step-by-Step Instructions
Firstly, one needs two .sdf files:
-    protein.sdf: containing all the coordinates of interest in the targeted protein structure
-    fragments.sdf: containing all the docked fragments
Both of .sdf files are derived                   from     BBB_first_iter.icb     (the    result   of   running
icm_load_hits_enamine_REAL.icm).
The table “BBB_first_iter” in the BBB_first_iter.icb has to be saved as fragments.sdf. The method for the
generation of the protein.sdf file is described below. Next, execute the executable CapSelection or
CapSelection2 (Available both for Windows and Linux (compiled for ubuntu 20.04/18.04)) in the same
folder with the protein.sdf and fragments.sdf files. The following command should be typed when
executing via terminal:
nohup ./CapSelection                 or nohup ./CapSelection2

The program outputs a file called CapSelect.sdf. As the result, the program adds four columns for all entries
of the fragments.sdf: “CapScores”, “Spheres”, “MaxMin” and “Distances”. The “CapScores” represents a
relative score indicating the potential of the fragment for growth; the “Spheres” column reports the number
of possible spheres; the “MaxMin” column contains a list of all minimum distances from a certain sphere
to the protein pocket; the “Distance” column gives the calculated distances from the center of all spheres to
the original coordinates of the cap.
After opening the CapSelect.sdf in ICM, the user needs to call the following command in the ICM terminal:
add column CapSelect
5.*Log(Abs(CapSelect.Score),2)+Log(Abs(CapSelect.CapScore),2) index=3
name="MergedScore" delete comment="Natural logarithm"

The fragments need to be sorted by the newly generated column ”MergedScore in descending order, the
first 1100 fragments should be selected for further enumeration. The selected pool of fragments exhibit
both a high docking score as well as a high CapScore.




Figure.2. Selection of fragments by MergedScore.
Upon successful execution of the CapSelection program, the user should proceed with the fragment
enumeration step.


Generation of the protein.sdf file.
The user saves those coordinates of the protein that are within the volume of interest defined by an external
box around maps (Figure 3, A). Thus, the volume should be slightly larger than the pocket and overall
docking maps.
Note: It is crucial to activate the “display hydrogens” option as well as “Toggle Wire Representation” by
clicking the corresponding menu on the left top corner of the ICM.




Next, the protein.sdf file can be generated by running the following commands in the ICM terminal:

String(Chemical( as_graph exact hydrogen                        ) mol)
# will highlight all corresponding atoms

followed by

add column t Chemical( as_graph exact hydrogen                           )
# will create table “t”

Lastly, the user needs to save the newly generated “t” table as protein.sdf file.
